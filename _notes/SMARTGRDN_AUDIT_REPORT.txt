7. Ka (SEASONAL/TEMPERATURE ADJUSTMENT FACTOR)
================================================================================

File: custom_components/smartgardn_et0/et0_calculator.py (lines 199-206)

SPECIFICATION MISMATCH DETECTED: ⚠️

AUDIT SPEC SAYS:
  Ka = 0.6 + 0.28 × (T/30) - 0.02 × (T/30)²
  Clamped to [0.4, 1.4]

IMPLEMENTATION:
  ka = 0.6 + 0.028 * t_max - 0.0002 * t_max**2

THESE ARE NOT EQUIVALENT!

COMPARISON AT T=10°C:
  Spec formula:   0.6 + 0.28×(10/30) - 0.02×(10/30)² = 0.6 + 0.0933 - 0.0022 = 0.691
  Implemented:    0.6 + 0.028×10 - 0.0002×10²         = 0.6 + 0.28 - 0.02 = 0.86
  Difference:     24% higher in implementation ❌

COMPARISON AT T=30°C:
  Spec formula:   0.6 + 0.28×(30/30) - 0.02×(30/30)² = 0.6 + 0.28 - 0.02 = 0.86
  Implemented:    0.6 + 0.028×30 - 0.0002×30²         = 0.6 + 0.84 - 0.18 = 1.26
  Difference:     47% higher in implementation ❌

IMPACT ON ETc:
ETc = ET0 × Kc × Ka

At T=30°C (summer):
  Using spec Ka=0.86:  ETc is reduced by 23%
  Using impl Ka=1.26:  ETc is increased by 48%
  Total discrepancy:   71% difference!

This significantly affects irrigation scheduling for warm periods.

STATUS: ⚠️ INCONSISTENCY BETWEEN SPECIFICATION AND IMPLEMENTATION

CLARIFICATION NEEDED:
1. Is the spec formula (from Geisenheimer data) the authoritative source?
2. Or is the implementation formula intentionally different?
3. Need to verify against original Ka literature/source

RECOMMENDATION:
Align implementation with specification or document the intentional difference
with scientific justification.
